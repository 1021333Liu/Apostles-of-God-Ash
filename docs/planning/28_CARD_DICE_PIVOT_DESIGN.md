# 卡牌骰子玩法转向设计

## 转向结论

《神烬使徒 / Apostles of God-Ash》的低语田野切片从实时动作房间战斗，转向叙事驱动的卡牌骰子决斗。

新的核心不是让玩家在田里追怪、躲危险区，而是让玩家进入一个被饥荒污染的场景，尝试与场景中央的人沟通。沟通失败后，双方进入一场像审讯、祭仪和搏斗混合在一起的牌局。战斗胜利后，对方才会说出真正的故事，并把日志碎片、记忆或卡牌奖励交给玩家。

这次转向保留：

- 低语田野世界观。
- 农夫、空腹者、稻草人、谷仓王等角色身份。
- 圣匣、日志碎片、第一故事拼接。
- 当前美术方向中的场景图、角色立绘、UI 器官感。

这次转向废弃或降级：

- 实时近战连段作为主玩法。
- 大量追逐、接触伤害、地面危险区。
- Hades-like 房间清怪节奏。

## 为什么必须转向

旧动作版的价值是证明 Godot 工程、低语田野场景、敌人身份、Boss、日志反馈和基础美术管线可以跑起来。但试玩后，它也暴露出几个关键困境：

1. 战斗好理解，但不够独特。近战、追怪、危险区和清房节奏容易落入普通动作原型。
2. 世界观很强，但玩法没有同等强。文案、圣匣、胃囊和日志碎片多数时候还是战斗后的说明，而不是战斗中的策略。
3. 日志系统应该是核心驱动力，但旧版里更像击杀后的掉落提示。
4. 玩家打赢怪物时，没有足够强烈地感觉自己是在“让对方说出故事，并把故事吃进身体里”。

所以新方向不是简单换类型，而是把项目最有辨识度的部分推到主循环里：

- 对话失败，就是进入战斗的理由。
- 攻击 / 防御牌，是玩家处理他人痛苦的方式。
- 骰子，是神之残骸对理解、抵抗和吞咽的判定。
- 胜利故事和日志碎片，是战斗的真正奖励。

## 高概念

你不是在杀死怪物。

你在用神之残骸与饥荒幸存者进行一场无法被语言解决的谈判。每一张牌都是一种处理他人痛苦的方式，每一次投骰都是神之残骸判断你是否有资格“吃下”这段记忆。

## 首版核心循环

```text
进入场景
-> 看到中央 NPC / 敌人
-> 尝试对话
-> 对方只吐出饥饿、孩子、名单、收割等碎片话语
-> 进入卡牌骰子决斗
-> 胜利后触发完整故事对话
-> 获得日志碎片
-> 三选一奖励
-> 回到地图 / 进入下一场景 / 回圣匣归档
```

## 场景结构

首版可以用当前场景美术做静态背景，不需要立即重做地图。

战斗前画面：

- 背景：低语田野入口、血肉麦田、谷仓、圣匣等。
- 中央：NPC 站在画面中央或略偏右。
- 左下：玩家角色小立绘或剪影。
- 下方：对话框。

卡牌战斗画面：

- 左侧：玩家。
- 右侧：敌人。
- 中央上方：敌人下一行动意图，例如“防御”“攻击”。
- 中央：骰子结算区，显示本次攻击骰、防御骰、伤害骰。
- 底部：手牌区。
- 左上或左下：玩家 HP、当前遗物/奖励。
- 右上或右侧：敌人 HP、行动序列预告。

胜利后画面：

- 背景压暗。
- 敌人不再攻击，开始讲述完整故事。
- 右侧或底部出现“样本归档 / 日志碎片获得”。
- 随后进入三选一奖励。

## 基础战斗规则

### 骰子

首版使用两个基础骰：

| 骰子 | 范围 | 用途 |
| --- | --- | --- |
| 命中骰 | 0-20 | 攻击命中、防御判定、对抗比较 |
| 效果骰 | 0-3 | 攻击伤害、反弹伤害、治疗或奖励浮动 |

说明：

- 这里的 `0-20` 是 21 个结果，不是传统 `1-20`。
- `20` 是大成功。
- `0` 是大失败。
- 后续道具可以加骰、重掷、扩大暴击收益或改变失败惩罚。

### 玩家基础牌

| 卡牌 | 类型 | 首版效果 | 成本 |
| --- | --- | --- | --- |
| 攻击 | Attack | 掷命中骰 `0-20`，若命中成立，再掷效果骰 `0-3` 造成伤害 | 1 |
| 防御 | Defense | 掷防御骰 `0-20`，用于抵挡敌人攻击；满足条件时反弹 | 1 |

MVP 建议每回合只打出 1 张牌。这样规则最清楚，也最适合先验证骰子对抗是否好玩。

正式版再扩展为：

- 每回合抽 3 张。
- 每回合 2 点行动力。
- 攻击、防御基础牌各消耗 1。
- 特殊牌可能消耗 0、2 或带副作用。

### 敌人行动

敌人不抽牌。每个敌人有固定行动序列。

首个农夫：

```text
防御 -> 攻击 -> 防御 -> 攻击 -> 循环
```

敌人每次行动也使用同样的命中骰和效果骰。

## 对抗结算

### 攻击对无防御

如果一方攻击，另一方没有防御：

- 攻击方掷命中骰。
- 命中骰 `0`：大失败，不造成伤害。
- 命中骰 `1-19`：造成效果骰 `0-3` 的伤害。
- 命中骰 `20`：大成功，固定造成 `3` 点伤害。

首版不再额外设置“闪避值”，避免玩家早期看不懂。

### 攻击对防御

如果一方攻击，另一方防御：

1. 攻击方掷命中骰。
2. 防御方掷防御骰。
3. 比较两者。

结算表：

| 条件 | 结果 |
| --- | --- |
| 防御骰是 `20` | 完美防御，必定无伤，并触发 `3` 点反弹 |
| 攻击骰是 `20`，防御骰不是 `20` | 攻击大成功，无法普通抵挡，固定造成 `3` 点伤害 |
| 攻击骰是 `0` | 攻击大失败，不造成伤害 |
| 防御骰 >= 攻击骰 + 5 | 防御成功并反弹，反弹伤害掷效果骰 `0-3` |
| 防御骰 >= 攻击骰 | 防御成功，无伤但不反弹 |
| 防御骰 < 攻击骰 | 防御失败，攻击方掷效果骰 `0-3` 造成伤害 |

### 防御对防御

双方都防御时，首版不做伤害。

建议表现：

- 双方都在观察。
- 玩家获得 1 个“归档进度”或“胃囊稳定”临时标记的可能性，后续再做。
- MVP 可以只显示“僵持”，直接进入下一回合。

### 大失败建议

首版大失败只做轻惩罚，不做严重扣血：

| 行动 | 掷出 0 的首版结果 |
| --- | --- |
| 攻击 | 挥空 / 理解失败，不造成伤害 |
| 防御 | 防御姿态崩坏，本次无法触发反弹 |

后续可以加入“胃囊躁动”“失去一张牌”“敌人获得意图强化”等代价。

## 初始数值

### 玩家

| 项目 | 首版值 | 理由 |
| --- | --- | --- |
| 最大 HP | 12 | 足够承受 4-6 次有效伤害，方便新手读懂规则 |
| 初始牌库 | 攻击 x5，防御 x5 | 经典基础牌库，利于后续做删牌和升级 |
| 每回合抽牌 | 3 | 让玩家通常能看到攻防选择 |
| 每回合出牌 | 1 | MVP 清晰，避免同时多张牌导致结算混乱 |
| 初始遗物 | 神之胃囊雏形 | 暂不加数值，只负责日志吸收和胜利归档 |

### 首个敌人：饥民农夫

| 项目 | 首版值 | 理由 |
| --- | --- | --- |
| HP | 9 | 平均 4-7 个交换回合结束 |
| 行动序列 | 防御、攻击、防御、攻击 | 玩家能预判，也能理解防御牌价值 |
| 攻击命中骰 | 0-20 | 与玩家一致，规则公平 |
| 攻击伤害骰 | 0-3 | 与玩家一致，方便读懂 |
| 防御骰 | 0-20 | 与玩家一致 |
| 反弹伤害骰 | 0-3 | 与玩家一致 |

### 首关目标体验

第一场农夫战应该达到：

- 玩家第一次能赢，但可能残血。
- 玩家能看懂“敌人下一回合要攻击，所以我要防御”。
- 玩家能经历至少一次反弹或大成功。
- 战斗时长约 3-5 分钟，包含对话和奖励。

## 概率与平衡说明

### 大成功与大失败

`0-20` 共 21 个结果。

| 事件 | 单次概率 |
| --- | --- |
| 掷出 20 | 约 4.76% |
| 掷出 0 | 约 4.76% |

大成功和大失败都不常见，但一场 6-10 次掷骰的战斗中通常能看见至少一次特殊结果。

### 防御反弹频率

当攻击和防御都不是特殊结果时，防御骰比攻击骰高 5 点以上才反弹。

粗略目标：

- 普通防御成功应常见。
- 反弹不应每次都有，否则攻击牌价值会下降。
- 反弹应该像“读懂敌人意图后的奖励”。

如果测试中反弹太强，可以把阈值从 `+5` 调到 `+6` 或 `+7`。

如果测试中防御太无聊，可以让普通防御成功时获得 1 点“胃囊稳定”，用于后续强化牌。

## 第一场农夫样板

### 战前对话

农夫不直接说完整剧情，只漏出碎片：

```text
天亮就下田。
名单别乱。
先收这片。
哭完再干活。
我好饿。
我的孩子还在等。
```

玩家无法通过普通对话解决，进入战斗。

### 战斗中短句

敌人攻击前：

```text
误了时辰，田会醒。
```

敌人防御前：

```text
规矩在这。
```

低血量：

```text
最好快点，天要亮了。
```

### 胜利后故事

胜利后农夫开始说更完整的话：

```text
我不是第一个把人带去田里的。
我只是记得路。

那年冬天，碗从门口摆到井边。
有人说，只要名单干净，粮就会回来。

我信了。
第二天我照常下田。
后来每天都是第二天。
```

随后掉落日志碎片，例如：

| 字段 | 示例 |
| --- | --- |
| id | `field_farmer_01` |
| title | `照常下田` |
| source_enemy | `farmer` |
| source_room | `field_gate` |
| text | `他不觉得自己在杀人。他只是害怕哪一天没有人继续照常。` |
| stomach_reaction | `胃囊收缩了一下，像在模仿劳动。` |

## 首关三选一奖励

击败第一关农夫后出现三选一。

| 奖励 | 类型 | 效果 | 设计意图 |
| --- | --- | --- | --- |
| 农夫的镰刀 | 攻击强化 | 所有攻击牌额外掷一次效果骰 `0-3`，作为追加伤害 | 直接、爽快、但波动大 |
| 农夫的帽子 | 防御强化 | 所有防御牌额外掷一次效果骰 `0-3`，加到防御骰结果上 | 让防御更容易反弹 |
| 农夫种的麦子 | 消耗 / 生命 | 立即掷 `1-3`，最大 HP 增加该数值，并治疗同等 HP | 给保守玩家续航 |

注意：

- “农夫的帽子”只增加防御点数，不把额外骰变成天然 20。
- “农夫的镰刀”的追加伤害不受基础伤害 `3` 上限限制，否则奖励体感太弱。
- “麦子”建议是最大 HP 和当前 HP 同时增加，避免玩家选了却还是快死。

## 日志系统的新位置

转向后，日志系统从“打小怪掉落”升级为“战后理解他人”的核心奖励。

首版规则：

- 每个敌人首次击败必掉一个主线日志碎片。
- 重复击败可以掉普通碎片或资源。
- 日志碎片在胜利后直接归档，不要求玩家在地图上捡取。
- 圣匣 UI 继续承担拼接第一故事的功能。

这样能避免卡牌战斗结束后还要回到动作地图里捡东西，节奏更干净。

## Godot MVP 实现方案

### 推荐架构

不要直接删除当前动作原型。首版建议新增一条卡牌模式路径：

```text
scenes/card_duel_demo.tscn
scripts/card/card_duel_controller.gd
scripts/card/card_database.gd
scripts/card/dice_resolver.gd
scripts/card/enemy_intent_runtime.gd
scripts/ui/card_hand_panel.gd
scripts/ui/dice_result_panel.gd
data/cards/basic_cards.json
data/encounters/low_field_farmer.json
```

### 状态机

`card_duel_controller.gd` 建议使用枚举状态：

```text
SCENE_INTRO
DIALOGUE_PRE
PLAYER_CARD_SELECT
RESOLVE_EXCHANGE
ENEMY_INTENT_ADVANCE
DIALOGUE_POST
REWARD_SELECT
LOG_ARCHIVE
COMPLETE
```

### 数据所有权

| 数据 | 所属 |
| --- | --- |
| 当前 HP、牌库、手牌、奖励 | `card_duel_controller.gd` 或后续 `RunState` |
| 卡牌定义 | `card_database.gd` / JSON |
| 敌人 HP、行动序列、台词、奖励池 | encounter JSON |
| 骰子规则 | `dice_resolver.gd` |
| 日志碎片归档 | 复用或扩展日志系统 |

### UI 节点草案

```text

├─ BackgroundLayer (TextureRect)

```

### 首版验收标准

- 能进入农夫场景并触发战前碎片对话。
- 玩家能在攻击牌和防御牌之间选择。
- 农夫按“防御、攻击、防御、攻击”循环行动。
- 骰子结果明确显示。
- 大成功、大失败、完美防御、反弹都能正确结算。
- 击败农夫后显示胜利故事。
- 出现三选一奖励，并能正确修改数值。
- 至少一个日志碎片进入圣匣或临时归档列表。

## 后续扩展方向

### 卡牌

- 追问：低伤害，但提高日志碎片质量。
- 咽下：造成伤害，同时获得胃囊躁动。
- 归档：防御并复制敌人一句话为日志。
- 断言：攻击骰低于 10 时可重掷一次。
- 空碗：若本回合没有造成伤害，治疗 1。

### 敌人

- 空腹者：攻击、攻击、防御，节奏更凶。
- 稻草人：防御、防御、尖鸣反弹，鼓励玩家用非攻击牌处理。
- 谷仓王：多阶段行动序列，能改变骰子规则。

### 胃囊系统

神之胃囊在新系统中不再是回血被动，而是卡牌构筑核心：

- 吃下记忆：把敌人的一句话变成临时牌。
- 过度消化：本场获得强力效果，但圣匣污染增加。
- 归档失败：死亡后保留部分日志，但失去部分奖励。

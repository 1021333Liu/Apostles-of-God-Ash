# 关卡 / HUD / 表现同学工作交接说明

这份文档给负责项目执行、关卡、HUD、表现和资产接入的技术同学使用。

## 结论先说

你不需要等正式美术资产。

当前阶段先用占位资产做关卡节奏、HUD 信息和战斗反馈。  
正式美术来了以后再替换。  
如果等美术齐了再开始做关卡，项目会卡住。

## 你的核心职责

你负责让原型从“能跑”变成“看得懂、测得出、能交付”。

你主要负责：

- 房间布局。
- HUD 信息。
- 危险区提示。
- 攻击 / 受击 / 击杀反馈。
- Boss 阶段提示。
- 资产接入准备。
- `CHANGELOG.md` 中和关卡、HUD、表现相关的更新记录。

## 推荐分支

使用分支：

```bash
git checkout feature/level-hud
```

每天开始：

```bash
git checkout main
git pull
git checkout feature/level-hud
git merge main
```

## 当前不要做

- 不要大改 `scripts/main.gd` 的核心战斗逻辑。
- 不要重写玩家、敌人、Boss 系统。
- 不要等正式美术。
- 不要一口气做十城邦。
- 不要先做复杂随机地图。

如果必须改 `scripts/main.gd`，先和负责核心玩法的人确认。

## 第一阶段任务

### 任务 1：房间布局占位

参考：

- `docs/planning/02_LEVEL_DESIGN.md`
- `docs/DEMO_VERTICAL_SLICE_SPEC.md`

先为这几个房间做布局草图或 Godot 占位：

1. 无声圣匣。
2. 低语田野入口。
3. 血肉麦田。
4. 肠道灌溉渠。
5. 饥饿谷仓。
6. 谷仓王胃室。

占位方式：

- 玩家：银白圆形或简单 Sprite。
- 空腹者：棕红色圆形。
- 饥民农夫：黄褐色圆形，加投掷提示。
- 饥饿稻草人：较大黄褐色形状，加横杆轮廓。
- 谷仓王：大型暗红形状，胃囊弱点用亮红。
- 危险区：半透明红色圆 / 椭圆。
- 可交互出口：金色或银色条形提示。

验收：

- 不看文档也能知道哪里危险。
- 房间之间看起来有变化。
- 玩家不会把敌人、危险区、出口混在一起。

### 任务 2：HUD 信息层级

HUD 至少包含：

- 当前房间名。
- 玩家 HP。
- 神之胃囊状态：可吞噬 / 闭合中 / 溢血刃。
- 当前目标：清怪 / 进入下一房间 / 击败 Boss。
- 交互提示：E / Enter。

验收：

- 战斗时不用读长句也知道目标。
- 胃囊能不能回血必须一眼看懂。
- HUD 不挡住玩家和敌人。

### 任务 3：战斗反馈

优先做：

- 攻击弧线。
- 受击闪烁。
- 伤害数字或短文本。
- 击杀回血提示。
- 胃囊闭合提示。
- Boss 阶段变化提示。

验收：

- 玩家知道自己有没有打中。
- 玩家知道自己为什么受伤。
- 玩家知道 Boss 进入新阶段。

### 任务 4：资产接入准备

先建立目录，不一定要有正式资源：

```text
assets/
  sprites/
    characters/
    enemies/
    bosses/
    environment/
    ui/
  audio/
    sfx/
    music/
```

命名建议：

```text
player_unrhymed_idle.png
enemy_empty_one_idle.png
enemy_farmer_throw.png
enemy_scarecrow_summon.png
boss_barn_king_phase1.png
ui_relic_stomach_open.png
ui_relic_stomach_closed.png
```

验收：

- 后续美术交付后可以直接放进对应目录。
- 程序接入时不用重新猜文件用途。

## 美术资产优先级

如果有美术支持，优先顺序是：

1. 玩家无韵回响。
2. 空腹者。
3. 危险地面。
4. 谷仓王。
5. 神之胃囊图标。
6. 饥民农夫。
7. 饥饿稻草人。
8. 圣匣背景。
9. 低语田野地面。

如果没有美术，继续用占位形状。  
当前目标是验证玩法，不是做最终画面。

## 每次提交要写更新日志

如果你改了关卡、HUD、表现、资产目录或反馈效果，都要更新：

`CHANGELOG.md`

格式可以很短：

```md
### 关卡表现：强化危险区提示

- 负责人：金荣俊
- 改动：调整危险区颜色和边缘提示。
- 原因：测试时玩家分不清装饰和伤害区。
- 影响范围：HUD / 关卡表现。
- 验证：进入血肉麦田后危险区可明显识别。
- 后续：接入正式血肉麦穗贴图。
```

## 本周完成标准

- 至少完成入口房、血肉麦田、谷仓王胃室的布局占位。
- HUD 能显示 HP、胃囊状态、房间名、目标。
- 危险区和 Boss 阶段提示比当前版本更清楚。
- 不破坏主场景运行。

# 《神烬使徒》画师绘制需求单

本文是给真人画师的直接任务单。目标不是继续出概念海报，而是画出能替换当前 Godot Demo 的战斗资产。

当前 Demo 已经能跑，但试玩暴露出几个明确问题：

- 角色和敌人多数只有一个侧面，左右移动时靠镜像，观感像“单边脸”。
- 程序临时翻面会有跳位风险，不能依赖代码长期修。
- 武器不能用一根程序线代替，必须画进攻击帧或单独给刀光特效。
- 背景图细节偏多，角色容易被地表纹理吞掉。
- 日志碎片需要像核心系统物件，而不是普通发光石头。

所以这份需求优先解决“战斗可读性 + 朝向 + 动作 + Godot 可接入”。

## 1. 总体美术标准

### 1.1 视角

所有战斗资产必须是：

- 16:9 横屏房间制动作肉鸽。
- 哈迪斯式 `3/4 斜俯视`。
- 能看到头肩、胸口、前后脚和身体厚度。
- 不是横版侧视。
- 不是正面立绘。
- 不是纯俯视圆饼。

画师可以把相机理解为：

```text
镜头从角色前上方约 35-45 度看下来。
角色站在同一个地面平面上。
脚底接触点要清楚。
身体不要画成横版格斗游戏那种侧面剪影。
```

### 1.2 风格

保留：

- 手绘厚涂。
- 暗黑奇幻。
- 宗教仪式感。
- 农业异化。
- 脏银、枯麦、旧布、封蜡、暗红胃纹。

降低：

- 露骨恐怖。
- 血肉堆叠。
- 恐怖电影海报感。
- 高复杂花纹。

允许：

- 更卡通、更清楚的比例。
- 更大的轮廓块面。
- 更少碎细节。
- 稍微夸张的动作姿态。

禁止：

- 像素风。
- 日系二次元描线。
- 3D 写实渲染。
- 科幻赛博 UI。
- 高饱和卡通。
- Q 版幼态。

## 2. 当前 Demo 的硬性接入要求

### 2.1 文件格式

- 最终交付：透明 PNG。
- 源文件：PSD / CSP / Krita 均可，但必须保留图层。
- 色彩：sRGB。
- 背景：角色、敌人、Boss、图标必须透明。
- 不要烘入投影到透明背景里；影子单独画或由 Godot 绘制。

### 2.2 画布和锚点

| 类型 | 单帧画布 | 角色落点 | 说明 |
| --- | ---: | --- | --- |
| 玩家 | `128x128` | 脚底中心在 `(64, 104)` 附近 | 允许武器和斗篷超出身体，但不能贴边 |
| 普通敌人 | `128x128` | 脚底中心在 `(64, 104)` 附近 | 和玩家同画布，便于碰撞 |
| 稻草人精英 | `160x160` | 脚底中心在 `(80, 132)` 附近 | 高剪影，需要上方空间 |
| Boss 谷仓王 | `384x384` | 底部中心在 `(192, 315)` 附近 | 大体积，但不能撑满画布 |
| 日志碎片 | `96x96` | 中心对齐 | 小尺寸仍可读 |
| 刀光 / 命中特效 | `192x192` 或 `256x256` | 中心对齐 | 透明 PNG 序列 |

验收时会在 Godot 里按当前大小缩放显示：

- 玩家约 `82px` 屏幕高度。
- 普通敌人约 `76-86px`。
- 稻草人约 `104px`。
- Boss 约 `260px`。

因此细节必须在缩小后仍能看清。

### 2.3 命名

当前接入可以继续使用单帧序列命名。后续会再整理成 spritesheet。

推荐命名：

```text
player_echo_idle_down_0.png
player_echo_idle_down_1.png
player_echo_walk_right_0.png
player_echo_attack_left_3.png
enemy_empty_walk_right_2.png
enemy_farmer_cast_down_4.png
boss_barn_king_phase2_attack_1.png
vfx_slash_echo_side_0.png
pickup_log_fragment_pulse_2.png
```

不要使用：

```text
角色最终版.png
新建画布2.png
怪物动作.png
boss改改.png
```

## 3. P0 第一批必须画

P0 的目标：替换当前 Demo 中最出戏的部分，让 5-8 分钟试玩成立。

### P0-1 玩家：无韵回响

用途：

- 当前试玩主角。
- 必须解决朝向、武器、攻击动作不协调的问题。

角色设定：

- 小体型斜俯视角色。
- 脏银白身体或盔甲。
- 胸口有暗红胃纹 / 胃囊印记。
- 武器不要画成长白棍；建议是短刃、残片刃、仪式骨刃或银色碎刃。
- 轮廓要比敌人更干净。

方向要求：

P0 先画 4 方向：

- `down`：面向屏幕下方，最常用展示方向。
- `up`：背向屏幕，能看到后肩和背部轮廓。
- `left`：面向左。
- `right`：面向右。

不要只画右侧再让程序镜像。玩家有胸口胃纹和武器，左右镜像会暴露问题。

动作帧数：

| 动作 | 每方向帧数 | 方向 | 说明 |
| --- | ---: | --- | --- |
| idle | 4 | down/up/left/right | 轻微呼吸，胸口胃纹微动 |
| walk | 6 | down/up/left/right | 脚底不滑，身体重心稳定 |
| attack_1 | 5 | down/up/left/right | 快速横斩或斜斩 |
| attack_2 | 5 | down/up/left/right | 第二段，幅度略大 |
| attack_3 | 6 | down/up/left/right | 重击，动作更明显 |
| hit | 3 | down/up/left/right | 受击后缩、胃纹变暗 |
| death | 6 | down | 首版只做一个方向即可 |

P0 可接受简化：

- 如果工期紧，先交 `down/left/right` 三方向，`up` 放 P1。
- 攻击三段可以先共用一套 `attack`，但必须有起手、挥出、收招。

验收标准：

- 缩到 82px 高时，能分辨头、胸口胃纹、武器方向。
- AD 移动不需要程序镜像也能看起来正常。
- 攻击帧里武器必须在手上，不允许由程序画一根线代替。
- 脚底接触点稳定，动画播放时角色不能左右跳位。

### P0-2 基础敌人：空腹者

用途：

- 第一房和多个房间的追击敌人。
- 主要负责“靠近、咬、被击退”的基础战斗反馈。

设定：

- 干瘪饥民。
- 空碗、喉咙、腹部失衡元素。
- 不要太恐怖，偏怪诞卡通。
- 轮廓比玩家更佝偻、更破碎。

方向要求：

- P0 画 `left/right/down`。
- `up` 可 P1。

动作帧数：

| 动作 | 每方向帧数 | 说明 |
| --- | ---: | --- |
| idle | 4 | 抽搐、空腹感 |
| walk | 6 | 追击步，脚步小而急 |
| attack | 4 | 前扑或张口咬 |
| hit | 3 | 被砍后后仰 |
| death | 5 | 倒下并散出记忆碎片小光 |

验收标准：

- 和玩家同屏时一眼能分辨。
- 攻击前 2-3 帧有明显前摇。
- 不能画成普通士兵。

### P0-3 中距离敌人：饥民农夫

用途：

- 第二、第三房核心敌人。
- 行为是播种危险区，不是普通近战战士。

设定：

- 农夫、播种、劳作姿态。
- 工具可以是破锄、种袋、登记牌。
- 动作要像“执行劳动”而不是挥刀打架。

方向要求：

- P0 画 `left/right/down`。
- `up` 可 P1。

动作帧数：

| 动作 | 每方向帧数 | 说明 |
| --- | ---: | --- |
| idle | 4 | 背着种袋或低头喘息 |
| walk | 6 | 拖步、护着种袋 |
| cast_seed | 6 | 播种前摇，必须明显 |
| hit | 3 | 被击中后种袋晃动 |
| death | 5 | 撒出残种和日志晶片光 |

验收标准：

- 播种动作在 1280x720 下必须清楚。
- 动作不像普通战士挥砍。
- 种袋或农具不能和背景麦草混在一起。

### P0-4 精英敌人：饥饿稻草人

用途：

- 第四房精英敌人。
- 释放麦浪 / 地面危险区。

设定：

- 稻草人、旧衣、木架、仪式符。
- 不要过度血肉化。
- 高瘦剪影，和普通敌人明显不同。

方向要求：

- P0 可只画 `down` 和 `side`。
- 该敌人可少移动，重点是施法可读性。

动作帧数：

| 动作 | 帧数 | 说明 |
| --- | ---: | --- |
| idle | 4 | 风吹草动 |
| cast_wave | 8 | 抬手 / 麦浪前摇 / 放出 |
| hit | 3 | 稻草散开 |
| death | 6 | 木架散落，掉日志碎片 |

验收标准：

- 施法前摇必须比普通攻击更明显。
- 身体高点不能贴画布上边。
- 在暗背景上轮廓仍清楚。

### P0-5 Boss：谷仓王

用途：

- 当前垂直切片 Boss。
- 必须有三阶段视觉差异。

设定：

- 谷仓、胃囊、王权、献祭制度融合。
- 是能在房间里战斗的大体积 Boss，不是横版背景里的巨大怪脸。
- 形体要有底座 / 足部 / 接地感。

阶段：

| 阶段 | 视觉状态 | 必须看见 |
| --- | --- | --- |
| phase1 | 仓门紧闭 | 王权符号、谷仓轮廓、压迫感 |
| phase2 | 仓门裂开 | 暗红裂缝、内部胃囊光 |
| phase3 | 红核暴露 | 弱点明确、攻击欲更强 |

动作帧数：

| 动作 | 每阶段帧数 | 说明 |
| --- | ---: | --- |
| idle | 4 | 大体积呼吸 / 仓门轻动 |
| move | 4 | 缓慢推进，可以很少位移 |
| attack_open | 6 | 仓门打开，召出空腹者 |
| attack_acid | 6 | 向地面吐出消化液 |
| hit | 3 | 弱点闪烁，身体震动 |
| phase_change | 8 | 阶段转换演出 |
| death | 10 | 王权崩塌，日志核心掉出 |

P0 可接受简化：

- `move` 可以先共用 `idle`。
- `death` 可以先只画 6 帧。
- 三阶段必须画，不能只改颜色。

验收标准：

- 260px 屏幕高度时仍能分清阶段。
- 红核弱点位置必须清楚。
- 不能遮住整个房间中心，Boss 要留战斗空间。

### P0-6 日志碎片 / 记忆晶片

用途：

- 击杀敌人后的叙事掉落物。
- 圣匣日志系统核心资产。

设定：

- 脏银晶片 + 油渍纸片 + 封蜡碎痕。
- 像“记忆样本”，不是金币、宝石或普通经验球。

尺寸：

- 原图 `96x96`。
- 游戏内显示约 `32-48px`。

状态：

| 状态 | 帧数 | 说明 |
| --- | ---: | --- |
| idle | 4 | 微弱浮动 |
| pickup_highlight | 4 | 可拾取时亮一下 |
| absorb | 6 | 被胃囊/圣匣吸收 |
| archive | 6 | 归档到纸页或封蜡 |

验收标准：

- 32px 下仍能识别为碎片。
- 不要像普通蓝水晶。
- 必须和 UI 里的日志图标同源。

## 5. 背景绘制需求

当前背景最大问题不是“不够好看”，而是角色在上面不够清楚。背景画师必须按战斗服务来画。

### 5.1 房间底图规则

- `1920x1080`。
- 3/4 斜俯视。
- 中心 60% 区域留作战斗空白。
- 地面纹理低对比，不要满地高频细节。
- 边界、障碍、危险区位置清楚。
- 前景遮挡不能盖住玩家和敌人。

### 5.2 五个房间需求

| 房间 | 文件名 | 画面重点 | 不要做 |
| --- | --- | --- | --- |
| 低语田野入口 | `bg_field_gate_base.png` | 枯麦边界、旧路牌、开阔入场 | 不要太暗，不要遮玩家 |
| 血肉麦田 | `bg_blood_wheat_base.png` | 红色危险区与麦田对比 | 不要满屏血肉 |
| 肠道灌溉渠 | `bg_gut_canal_base.png` | 三条可读渠线，边缘清楚 | 不要让渠线像装饰花纹 |
| 饥饿谷仓 | `bg_hungry_barn_base.png` | 谷仓门、稻草人仪式感 | 不要横版谷仓立面 |
| 谷仓王胃室 | `bg_barn_king_base.png` | Boss 场地、弱点暴露区域 | 不要让 Boss 和背景粘在一起 |

### 5.3 背景验收

把玩家、空腹者、农夫放在背景上看：

- 玩家必须 1 秒内被找到。
- 敌人不能和麦草同色。
- 红色危险区不能和纯装饰红斑混淆。
- 房间边界要知道哪里能走，哪里不能走。

## 6. UI 绘制需求

UI 风格是“圣匣档案”，不是普通动作游戏 HUD。

### 6.1 P0 UI

| 资产 | 尺寸 | 格式 | 说明 |
| --- | ---: | --- | --- |
| 生命条底板 | `320x40` | PNG / NinePatch | 脏银金属边，暗红填充 |
| 胃囊四态图标 | `96x96` x 4 | PNG | 开启、闭合、溢血、归档 |
| 日志碎片小图标 | `96x96` | PNG | 和掉落物同源 |
| 拾取提示底板 | `360x72` | NinePatch PNG | 油渍纸条 + 封蜡角 |
| 圣匣日志面板概念 | `1920x1080` | PSD + PNG | 左碎片列表，右纸页文本，底部真相进度 |

### 6.2 UI 禁止项

- 霓虹蓝紫。
- 科幻扫描线。
- 玻璃拟态。
- 现代手机 App 卡片感。
- 把大量文字画死在图片里。

## 9. 验收红线

以下情况直接退回重画：

- 角色是横版侧视。
- 角色是正面立绘，不能当战斗精灵。
- 只有一个侧面，全靠程序镜像。
- 动画脚底跳位。
- 武器漂浮或从身体中心长出来。
- 背景比角色更抢眼。
- 日志碎片像普通宝石或金币。
- Boss 只是普通怪物放大。
- 画面太血腥、太恐怖，不符合“卡通化暗黑奇幻”。

## 10. 程序接入备注

画师交付后，程序需要做两件事：

1. 把当前 `art_asset_registry.gd` 从“单状态帧组”升级为“动作 + 方向”帧组。
2. 玩家和敌人的朝向逻辑从左右镜像改为读取 `down/up/left/right`。

所以画师现在必须按方向命名。不要等程序改完再画，否则又会回到单侧脸问题。

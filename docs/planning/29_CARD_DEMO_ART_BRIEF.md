# 卡牌骰子 Demo 美术需求单

## 目标

本文件用于把《神烬使徒 / Apostles of God-Ash》的新方向做成可试玩 Demo。

当前玩法已经从实时动作战斗转向“叙事场景 + 卡牌化行动选择 + 骰子对抗”。因此第一批美术不再优先做四方向跑动和攻击动画，而是优先做：

- 场景背景。
- 玩家与敌人对峙动画组。
- 攻击 / 防御卡牌图面。
- 骰子和结算 UI。
- 胜利后的日志碎片与奖励 UI。

画师目标不是画宣传图，而是画能直接放进 Godot Demo 的 2D 资产。

## 美术总方向

### 气质

- 低饱和乡村荒凉。
- 干燥、安静、冷白光。
- 枯麦、灰褐土地、旧纸、脏银、封蜡、木板。
- 暗红只作为系统重点：胃囊、伤害、危险、日志核心、封蜡。

可以参考 Andrew Wyeth 作品里的乡村孤独、干燥材质、低饱和空旷感，但只作为气质规则：

- 不临摹具体作品。
- 不复刻构图。
- 不写“某某画家风格”作为唯一指令。
- 不做纯写实油画海报。

### 游戏可读性

所有图进入游戏后需要服务 16:9 横屏界面。

优先级：

1. 玩家和敌人对峙关系。
2. 敌人意图：攻击 / 防御。
3. 玩家可选行动：攻击 / 防御。
4. 骰子结果。
5. 日志碎片和胜利故事。
6. 背景氛围。

如果背景漂亮但抢 UI，退回修改。

## P0 必须资产

P0 是 demo 能跑出来、能给同学试玩、能让人理解新方向的最低资产包。

### P0-1 战斗背景：低语田野入口

用途：

- 第一场农夫战的背景。
- 玩家进场、战前对话、卡牌决斗都用这一张。

规格：

| 项目 | 要求 |
| --- | --- |
| 文件名 | `bg_card_field_entrance.png` |
| 尺寸 | `1920x1080` |
| 格式 | PNG |
| 视角 | 16:9 横屏，轻 3/4 斜俯视或叙事背景 |
| 中心 | 必须留出玩家和农夫立绘空间 |
| 下方 | 必须适合叠放卡牌 UI |

画面内容：

- 枯麦田入口。
- 灰褐干土。
- 旧木牌，上面有“丰收”痕迹和名单刻痕。
- 少量空碗、谷袋、破农具。
- 中央不要堆满细节。

禁止：

- 横版平台关卡。
- 纯写实风景照片感。
- 满屏麦草纹理。
- 红色装饰太多，抢伤害 / 日志重点。

### P0-2 玩家对峙动画组：无韵回响

用途：

- 战斗时站在左侧。
- 默认站姿不是静态贴图，需要有呼吸、胃纹收缩、衣物微动。
- 使用攻击、防御、受击和胜利结算时触发对应短动画。

规格：

| 项目 | 要求 |
| --- | --- |
| 文件名规则 | `actor_player_echo_<action>_<frame>.png` |
| 单帧尺寸 | 建议 `768x768` |
| 格式 | 透明 PNG 序列，另交源文件 |
| 姿态 | 3/4 对峙姿态，面向右侧敌人 |
| 游戏内显示 | 约 `260-360px` 高 |

视觉关键词：

- 脏银白身体或盔甲。
- 胸口有暗红胃囊纹。
- 短刃、残片刃或仪式骨刃。
- 小体型，但站姿稳定。
- 轮廓干净，比敌人更容易追踪。

P0 至少交 5 个动作：

| 动作 | 帧数 | 文件名示例 | 用途 |
| --- | ---: | --- | --- |
| idle | 6 | `actor_player_echo_idle_0.png` - `_5.png` | 默认对峙循环，呼吸、胃纹轻微收缩 |
| attack | 8 | `actor_player_echo_attack_0.png` - `_7.png` | 使用攻击牌时播放，起手、挥出、收招清楚 |
| defend | 6 | `actor_player_echo_defend_0.png` - `_5.png` | 使用防御牌时播放，胃囊闭合或残片护身 |
| hit | 4 | `actor_player_echo_hit_0.png` - `_3.png` | 受到伤害时播放，胸口胃纹变暗 |
| victory | 6 | `actor_player_echo_victory_0.png` - `_5.png` | 胜利后采样 / 归档时播放 |

P0 如果时间很紧，最低可先交：

- `idle` 6 帧。
- `attack` 6 帧。
- `defend` 4 帧。

但不能只交一张静态站姿。角色必须在战斗过程中有可见动效。

禁止：

- 正面证件照。
- 横版侧面格斗角色。
- 复杂披风遮住身体。
- 武器像白色长棍。
- idle 完全不动，像纸片贴在背景上。

### P0-3 敌人对峙动画组：饥民农夫

用途：

- 第一场战斗的核心敌人。
- 战前咕噜话、战斗、胜利后故事都围绕他。
- 默认站姿要有喘息、种袋晃动、低头碎语。
- 攻击和防御意图必须通过动画提前读出来。

规格：

| 项目 | 要求 |
| --- | --- |
| 文件名规则 | `actor_enemy_farmer_<action>_<frame>.png` |
| 单帧尺寸 | 建议 `768x768` |
| 格式 | 透明 PNG 序列，另交源文件 |
| 姿态 | 3/4 对峙姿态，面向左侧玩家 |
| 游戏内显示 | 约 `280-380px` 高 |

视觉关键词：

- 饥民农夫。
- 旧帽子、破布、种袋、镰刀或锄。
- 像劳动者，不像正规战士。
- 有“饿”和“照常下田”的矛盾感。
- 怪诞，但不要纯恐怖。

P0 至少交 6 个动作：

| 动作 | 帧数 | 文件名示例 | 用途 |
| --- | ---: | --- | --- |
| idle | 6 | `actor_enemy_farmer_idle_0.png` - `_5.png` | 默认对峙循环，喘息、低头、种袋轻晃 |
| mutter | 6 | `actor_enemy_farmer_mutter_0.png` - `_5.png` | 战前碎片对话，嘴部 / 头部小动 |
| attack | 8 | `actor_enemy_farmer_attack_0.png` - `_7.png` | 农夫攻击回合，镰刀或农具前摇明显 |
| defend | 6 | `actor_enemy_farmer_defend_0.png` - `_5.png` | 农夫防御回合，护住种袋 / 压低帽檐 |
| hit | 4 | `actor_enemy_farmer_hit_0.png` - `_3.png` | 被击中时播放，后退但不飞出画布 |
| confess | 8 | `actor_enemy_farmer_confess_0.png` - `_7.png` | 胜利后讲故事，姿态塌下 |

禁止：

- 画成普通僵尸。
- 画成帅气镰刀战士。
- 背景和身体都是同一种麦黄色，看不清。
- 只有一张站姿，靠 UI 表示攻击 / 防御。

### P0-4 基础卡牌：攻击 / 防御

用途：

- 玩家每回合固定二选一。
- 首版不做抽牌，但视觉仍按卡牌按钮制作。

规格：

| 项目 | 要求 |
| --- | --- |
| 单卡尺寸 | `512x768` |
| 格式 | PNG，建议保留 PSD |
| 文字 | 不要把正文效果画死；卡名可画，也可留给程序 |
| 风格 | 旧纸卡面 + 脏银边框 + 封蜡 / 胃纹系统点缀 |

#### 攻击牌

文件名：

```text
card_basic_attack_art.png
```

画面：

- 脏银短刃或神烬残片划过旧纸。
- 暗红胃纹残影。
- 不要大血喷溅。
- 卡面一眼能读作攻击。

#### 防御牌

文件名：

```text
card_basic_defense_art.png
```

画面：

- 金属夹、封蜡、旧纸护符或胃囊闭合纹。
- 视觉要像“封住 / 抵挡 / 归档失败样本”。
- 不要画成普通盾牌图标就完事。

卡牌 UI 需要留区域：

- 顶部卡名。
- 中央图面。
- 底部效果文字。
- 左上角行动力或行动标识。

### P0-5 骰子与结算 UI

用途：

- 显示攻击骰、防御骰、效果骰。
- 让玩家理解这套系统不是随机扣血，而是可读的仪式判定。

规格：

| 资产 | 尺寸 | 文件名 |
| --- | ---: | --- |
| 命中骰图标 | `128x128` | `ui_die_hit_d20.png` |
| 防御骰图标 | `128x128` | `ui_die_defense_d20.png` |
| 效果骰图标 | `128x128` | `ui_die_effect_d3.png` |
| 大成功标记 | `256x128` | `ui_roll_critical_success.png` |
| 大失败标记 | `256x128` | `ui_roll_critical_fail.png` |
| 完美防御标记 | `256x128` | `ui_roll_perfect_defense.png` |
| 反弹标记 | `256x128` | `ui_roll_reflect.png` |
| 骰子结果底板 | `640x180` | `ui_dice_result_panel.png` |

视觉：

- 骰子可以是骨质、脏银、旧牙、封蜡刻面。
- 数字由程序渲染，不要画死 `20` 或 `0`。
- 大成功可以用暗红 + 脏银光。
- 大失败不能太搞笑，应该像“胃囊拒绝理解”。

禁止：

- 现代桌游塑料骰。
- 霓虹 D20。
- 科幻 HUD。


数据里只引用这些稳定路径，不要引用画师源文件路径。

Godot 导入设置：

- UI、卡牌、骰子：Lossless，Filter On 或按项目观感决定。

## 动画接入规则

角色动画不要交视频文件。首版统一交透明 PNG 序列，程序用 Godot 的 `AnimatedSprite2D`、`Sprite2D` 帧切换或 `AnimationPlayer` 播放。

推荐动画状态机：

```text
idle_loop
-> player_attack_once
-> idle_loop

idle_loop
-> player_defend_once
-> idle_loop

idle_loop
-> hit_once
-> idle_loop

enemy_idle_loop
-> enemy_mutter_loop / enemy_attack_once / enemy_defend_once / enemy_hit_once / enemy_confess_loop
```

播放约定：

| 事件 | 玩家动画 | 农夫动画 |
| --- | --- | --- |
| 战前对话 | `idle` | `mutter` |
| 玩家选攻击 | `attack` | 若农夫本回合防御则 `defend`，否则 `idle` |
| 玩家选防御 | `defend` | 若农夫本回合攻击则 `attack`，否则 `defend` |
| 玩家受伤 | `hit` | `attack` 收招 |
| 农夫受伤 | `attack` 收招 | `hit` |
| 胜利故事 | `victory` 或 `idle` | `confess` |

帧率建议：

| 动作 | FPS |
| --- | ---: |
| idle | 6 |
| mutter | 6 |
| attack | 12-14 |
| defend | 8-10 |
| hit | 12 |
| victory / confess | 6-8 |

锚点要求：

- 每一帧角色脚底或身体底部中心必须稳定。
- 攻击时可以前倾，但不要让整个人在画布里大幅漂移。
- 武器、帽子、种袋可以动，但不能贴边或被裁切。
- 所有动作同一个角色必须保持相同画布尺寸和同一锚点。

## 程序 Demo 依赖关系

美术到位前，程序可以先用色块和文字占位实现 `CardDuelDemo`。

美术到位后按这个顺序接：

1. 背景。
2. 玩家 / 农夫对峙动画组。
3. 攻击 / 防御卡牌。
4. 敌人意图图标。
5. 骰子结果 UI。
6. 胜利故事纸页。
7. 日志碎片和奖励三选一。

这样即使美术只交一半，Demo 仍然能逐步替换占位。

## 验收红线

以下情况不能进 Demo：

- 立绘没有透明背景。
- 角色只是一张静态图，没有 idle / attack / defend 动画。
- 背景中心太花，卡牌和骰子看不清。
- 卡牌把效果文字画死，程序无法换文本。
- 农夫像普通战士或僵尸，失去“劳动者”身份。
- 骰子和 UI 变成科幻蓝光。
- 日志碎片像普通宝石。
- 奖励图标像通用 RPG 装备。
- 整体太血腥，压过“饥荒制度”和“档案归档”的叙事。

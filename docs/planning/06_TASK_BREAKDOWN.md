# 制作任务拆分与里程碑

## 当前阶段

当前项目已经有一个可运行的单脚本原型。下一阶段不是继续堆功能，而是把它拆成可制作、可替换资产、可调参的垂直切片。

## 里程碑 1：原型可玩

目标：保持当前程序化原型，但把战斗节奏调到能连续测试。

任务：

| 优先级 | 任务 | 负责人类型 | 完成标准 |
| --- | --- | --- | --- |
| P0 | 调整攻击范围、冷却、伤害 | 数值/程序 | 近战命中稳定，不需要贴脸 |
| P0 | 调整空腹者速度和 HP | 数值 | 入口房 40-60 秒完成 |
| P0 | 调整胃囊闭合和溢血反馈 | 系统/数值 | 玩家能主动利用击杀回血 |
| P1 | 给 Boss 阶段加更明显提示 | 程序/UI/特效 | 阶段变化一眼可见 |
| P1 | 加死亡原因记录 | 程序/文案 | 回圣匣文本能反映死因 |

验收：

- Godot 启动无错误。
- 一轮能从圣匣打到 Boss 或死亡回收。
- 至少 3 名测试者能理解操作和目标。

## 里程碑 2：结构拆分

目标：把单脚本原型拆成可长期维护的 Godot 场景结构。

建议结构：

| 场景/脚本 | 责任 |
| --- | --- |
| `Player.tscn` / `player.gd` | 移动、攻击、受击、朝向 |
| `EnemyBase.tscn` / `enemy_base.gd` | HP、接触伤害、死亡信号 |
| `EmptyOne.tscn` | 空腹者 |
| `FamineFarmer.tscn` | 饥民农夫 |
| `HungryScarecrow.tscn` | 饥饿稻草人 |
| `BarnKing.tscn` | Boss 阶段和技能 |
| `RoomController.gd` | 房间加载、完成检测 |
| `RelicSystem.gd` | 胃囊、溢血、闭合 |
| `HUD.tscn` | HP、房间名、残骸状态、提示 |

验收：

- 功能与单脚本原型等价。
- 每个实体能单独替换美术。
- 数值集中在表或导出变量中。

## 里程碑 3：首版美术替换

目标：用低成本但风格明确的资产替换几何占位。

任务：

| 优先级 | 资产 | 完成标准 |
| --- | --- | --- |
| P0 | 玩家无韵回响 | 移动和攻击时可读 |
| P0 | 空腹者 | 与玩家明显区分 |
| P0 | 危险地面 | 范围和伤害预警清楚 |
| P0 | 谷仓王 | 胃囊弱点明确 |
| P1 | 圣匣背景 | 与田野形成情绪对比 |
| P1 | 房间地面 | 体现血肉麦田 |
| P2 | 收藏家半身像 | 支撑文本呈现 |

验收：

- 截图可用于内部展示。
- 玩家不会把危险地面误认为装饰。
- Boss 弱点在阶段三足够醒目。

## 里程碑 4：叙事闭环

目标：让死亡、击败 Boss 和残骸获得成为叙事推进，而不是普通结算。

任务：

| 优先级 | 任务 | 完成标准 |
| --- | --- | --- |
| P0 | 开场收藏家文本 | 30 秒内建立目标 |
| P0 | 死亡回收文本池 | 至少 6 条，按死因或房间变化 |
| P0 | Boss 开场/阶段/死亡文本 | 每阶段 1-2 条 |
| P1 | 静默者晶片界面 | 能显示至少 5 条晶片文本 |
| P1 | 胃囊记忆结算 | 击败 Boss 后显示关键记忆 |

验收：

- 玩家能复述“谷仓王为什么变成这样”。
- 玩家能理解胃囊既是能力也是侵蚀。
- 文本不阻塞战斗节奏。

## 里程碑 5：垂直切片评审版

目标：形成可以给外部朋友或小范围玩家试玩的版本。

必须包含：

- 5-8 分钟完整流程。
- 可识别首版美术。
- 基础音效和打击反馈。
- 完整死亡回收。
- 完整 Boss 胜利结算。
- 一份测试问卷或反馈表。

评审问题：

1. 你是否愿意再来一局？
2. 你是否理解胃囊的收益和代价？
3. 你是否记得谷仓王？
4. 哪个房间最烦？
5. 哪个反馈最爽？
6. 你觉得它像普通肉鸽，还是有自己的主题？

## 当前不做

- 十城邦全量。
- 复杂随机地图。
- 多武器库。
- 长篇对话。
- 商店系统。
- 永久养成树。
- 多结局。

这些内容可以进长期路线，但不进入低语田野首版。

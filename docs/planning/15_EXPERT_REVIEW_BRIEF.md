# 《神烬使徒》专家评审简报

本文档用于向游戏行业、美术、叙事设计或独立游戏制作方向的专家说明《神烬使徒 / Apostles of God-Ash》的当前方案，并征求评审意见。

当前项目处于 Godot 可玩原型阶段。我们希望在下一阶段确认两件事：

1. 美术方向是否成立，是否适合小团队落地。
2. 日志碎片系统是否能成为游戏的核心叙事与成长循环。

## 1. 项目一句话

《神烬使徒》是一款黑暗奇幻动作肉鸽。玩家扮演“无韵回响”，进入被旧神残骸污染的城邦，击败使徒，收集神之残骸，并用这些残骸拼装自己的战斗能力与人格。

首个垂直切片聚焦“低语田野”：一片被饥荒、献祭制度和暴食回响改造成活体农田的区域。玩家需要击败“谷仓王”，获得残骸“神之胃囊”，并通过掉落的日志碎片拼出低语田野的第一段真相。

## 2. 当前垂直切片内容

当前 Demo 目标是 5-8 分钟可玩切片，而不是完整商业版本。

已实现或正在实现的核心内容：

- 无声圣匣：玩家复活、出发、归档残骸的起点。
- 低语田野：固定房间链，包括入口、血肉麦田、肠道灌溉渠、饥饿谷仓、谷仓王胃室。
- 基础战斗：俯视角近战、三段攻击、敌人接触伤害、危险地面。
- 敌人类型：空腹者、饥民农夫、饥饿稻草人、谷仓王。
- 残骸机制：神之胃囊提供击杀回血、受击闭合、满血溢出转攻击。
- 世界观反馈 UI：房间档案、击杀采样、死亡回收、Boss 弱点窗口。
- HUD 初版：生命条、胃囊状态条、记忆晶片图标、像素风暗黑面板。

## 3. 美术方向方案

### 3.1 核心判断

我们希望美术方向接近“古典油画厚涂 + 暗黑奇幻 + 舞台化光影”的质感，但不追求 3A 级 3D 写实规格。

更准确的落地方向是：

> 2D 手绘油画感动作肉鸽。

也就是说，Godot 负责承载 2D 动作、UI、粒子、Shader 和分层背景；美术通过厚涂角色、手绘怪物、笔触背景和材质化 UI 建立气质。

### 3.2 不做什么

为了控制团队规模和制作风险，本项目当前不做：

- 3D 写实角色。
- 电影级实时灯光。
- 高精度面部表情。
- 大规模开放地图。
- 复杂随机地牢。
- 大量武器和装备库。

这些内容成本高，且会稀释当前最重要的“低语田野垂直切片”。

### 3.3 视觉关键词

建议美术关键词：

- 厚涂。
- 油彩笔触。
- 暗红血肉。
- 枯金麦田。
- 骨白残骸。
- 脏银神血。
- 黑绿腐败。
- 戏剧舞台光。
- 档案残页。
- 胃液浸泡的画布。

### 3.4 色彩语言

| 系统 | 主色 | 作用 |
| --- | --- | --- |
| 无声圣匣 | 冷灰、骨白、脏银 | 安静、回收、实验、神性残骸 |
| 低语田野 | 枯金、暗褐、腐红 | 饥荒、农业、制度化暴食 |
| 神之胃囊 | 暗红、肉粉、黑紫 | 生命转换、吞噬、代价 |
| 记忆晶片 | 半透明银白、冷青 | 叙事碎片、真相、归档 |
| Boss 弱点 | 高饱和红、金色裂光 | 战斗窗口和仪式感 |

### 3.5 角色和敌人美术需求

| 对象 | 视觉定位 | 首版需求 |
| --- | --- | --- |
| 无韵回响 | 无固定人格的容器，银白身体，胸口胃纹 | 俯视/斜俯视待机、移动、攻击、受击、死亡 |
| 空腹者 | 干瘪饥民，喉咙和腹部失衡 | 移动、扑咬、死亡 |
| 饥民农夫 | 仍在劳动的献祭执行者 | 投掷/播种前摇、受击、死亡 |
| 饥饿稻草人 | 旧命令和田间警戒装置 | 召唤麦浪、尖鸣、受击 |
| 谷仓王 | 谷仓、胃、王权和献祭制度的集合体 | 三阶段、胃囊暴露、死亡演出 |
| 收藏家 | 医生、档案员、研究者，而非直接恶魔 | 半身立绘或场景剪影 |

### 3.6 UI 美术方向

UI 不应是科幻仪表，也不应是普通血条。

建议 UI 语言：

- 圣匣档案。
- 旧医学记录。
- 被油和胃液浸泡的纸页。
- 手写批注。
- 残骸槽。
- 封蜡、针脚、金属夹。
- 图标像“被收藏的器官标本”。

关键 UI：

- 生命条：暗红液体 + 脏银边框。
- 胃囊状态：开启、闭合、溢血、归档四态。
- 日志碎片：破损画布或晶片残页。
- 房间档案：异常报告，而不是普通任务提示。
- 死亡回收：失败样本档案，而不是 Game Over。

## 4. 系统方案：日志碎片系统

### 4.1 为什么需要日志系统

当前项目最大的叙事潜力不在长对话，而在“玩家打怪后获得被吞噬者的记忆碎片”。

我们希望把碎片化叙事做成核心系统，而不是背景设定：

> 打小怪不是清杂兵，而是在采样世界。  
> 收集日志不是看百科，而是在拼回一个被制度吃掉的故事。

### 4.2 核心循环

建议下一阶段实现以下 MVP：

1. 玩家击杀敌人。
2. 敌人有概率掉落日志碎片。
3. HUD 短暂显示“采样记录”。
4. 碎片进入圣匣档案。
5. 玩家在圣匣 UI 查看“低语田野日志”。
6. 收集关键碎片后，系统拼出第一段完整故事。
7. 这段故事反过来改变玩家对谷仓王、田野和神之胃囊的理解。

### 4.3 日志碎片来源

| 来源 | 掉落内容 | 叙事作用 |
| --- | --- | --- |
| 空腹者 | 饥民、粮票、被排除者记忆 | 说明普通居民如何变成空腹者 |
| 饥民农夫 | 献祭执行、邻里关系、产量记录 | 说明制度如何让杀人变成劳动 |
| 饥饿稻草人 | 旧命令、警戒名单、逃祭者记录 | 说明恐惧如何被自动化 |
| 精英房 | 静默者晶片 | 说明村民不是单纯受害者 |
| 谷仓王 | 神之胃囊核心记忆 | 说明 Boss 曾经真的试图阻止饥荒 |
| 死亡回收 | 玩家失败样本 | 说明土地正在学习玩家 |

### 4.4 第一组故事主题

第一组日志应拼出的问题是：

> 低语田野为什么开始吃人？

建议第一段完整故事：

低语田野原本是饥荒中的村庄。谷仓王起初打开粮仓救人，后来粮食不够，村民开始接受献祭。献祭一开始针对外乡人、罪犯和“可以被牺牲的人”，后来扩大到每一户。谷仓王献出自己的胃，试图替所有人承受饥饿。血肉麦穗真的长出来后，制度获得了成果，于是没人再愿意停止。最后，饥荒没有被解决，只是学会了用谷仓王的嘴说话。

### 4.5 日志系统结构

建议数据结构：

```gdscript
{
	"id": "field_farmer_03",
	"area": "low_whispering_field",
	"source": "farmer",
	"title": "本季收成稳定",
	"fragment": "后来没人再问该不该献。我们只问轮到哪户。",
	"story_key": "field_origin",
	"order": 3,
	"rarity": "common"
}
```

### 4.6 圣匣日志 UI

圣匣中应新增一个“低语田野日志”界面。

首版 UI 不需要复杂：

- 左侧：日志碎片列表。
- 右侧：当前选中碎片文本。
- 底部：故事拼合进度，例如 `低语田野真相 3/5`。
- 收集齐关键碎片后：出现完整档案“谷仓王与第一次开仓”。

重要的是：玩家能知道自己正在拼故事，而不是随机捡文本。

## 5. 美术和系统的结合方式

美术不只是包装，必须服务系统。

| 系统 | 美术表达 |
| --- | --- |
| 击杀掉落日志 | 敌人死亡时掉出银白晶片或残页 |
| 胃囊吞噬 | 胸口红色胃纹张开，碎片被吸入 |
| 胃囊闭合 | 胃囊图标闭合，角色胸口变暗 |
| 溢血转刃 | 红银油彩笔触沿武器扩散 |
| 圣匣归档 | 日志碎片被夹入档案页 |
| Boss 弱点 | 谷仓王腹部像画布被撕开，露出红核 |

## 7. 想请专家重点判断的问题

### 美术方向

1. “2D 手绘油画感动作肉鸽”是否比像素风更适合这个题材？
2. 以小团队规模，厚涂油画感是否可控？是否应改为“油画感立绘 + 简化俯视角色”的折中方案？
3. 低语田野的主视觉是否应更强调农业恐怖、宗教仪式，还是神性残骸？
4. UI 做成圣匣档案和医学标本风，是否能强化差异化？

### 系统方向

1. 日志碎片系统是否足以成为本作的核心叙事系统？
2. “打小怪掉落记忆，拼出第一段故事”是否能支撑玩家继续探索？
3. 日志碎片应该只影响叙事，还是应与能力解锁、Boss 弱点、结局倾向产生关系？
4. 圣匣 UI 应该在每次死亡后强制展示，还是让玩家主动查看？

### 商业与制作判断

1. 这个题材是否有足够清晰的卖点？
2. 当前垂直切片是否应继续打磨战斗手感，还是先建立美术样张和日志系统？
3. 如果参加学生作品、独立游戏展或 Demo 评审，最应该先补哪一块？
4. 以当前团队规模，是否应压缩 Boss、房间或敌人数量？

## 8. 我们当前的下一步建议

下一阶段不建议继续扩展十城邦，而是完成低语田野的“可展示核心闭环”：

1. 做日志碎片系统 MVP。
2. 做圣匣日志 UI。
3. 接入第一组低语田野日志文本。
4. 做一张油画风视觉基准图。
5. 做玩家、空腹者、饥民农夫的首版手绘占位精灵。
6. 把击杀掉落、胃囊吞噬、日志归档做成可见反馈。

完成后，Demo 的表达应从“能打怪的原型”提升到：

> 玩家在一片吃人的田里战斗，并通过敌人掉落的记忆碎片，拼出这片田为什么变成怪物。

# 《神烬使徒》美术细节需求附录

本文是 `22_ARTIST_DRAWING_BRIEF.md` 的细化版，用于美术负责人拆分任务、画师自检、程序接入前验收。

核心目标：

- 让当前 Godot Demo 不再像“概念图硬塞进游戏”。
- 让角色、敌人、武器、刀光、危险区、日志碎片和 UI 都服务房间制动作肉鸽。
- 让画师知道每个细节为什么要画、画到什么程度、怎样才算能进游戏。

## 1. 画面优先级

战斗画面里，视觉优先级固定如下：

1. 玩家位置。
2. 敌人攻击前摇和危险区。
3. 玩家攻击范围和命中反馈。
4. 日志碎片掉落。
5. 背景氛围。
6. 装饰细节。

如果背景、笔触、纹理、光影影响前 4 项，一律减弱。

## 2. 屏幕读图规则

### 2.1 1280x720 验收截图

所有资产必须在 `1280x720` 截图里测试，而不是只看原画大图。

验收截图至少包含：

- 玩家站在房间中心。
- 玩家和 2 个空腹者同屏。
- 玩家和 1 个农夫 + 2 个危险区同屏。
- 玩家攻击敌人的瞬间。
- 敌人掉落日志碎片的瞬间。
- Boss phase1 / phase2 / phase3 同场景对比。

失败标准：

- 玩家需要找 2 秒以上才能定位。
- 敌人和麦草颜色混在一起。
- 红色危险区和装饰血迹无法区分。
- 日志碎片像普通宝石、金币、经验球。
- Boss 弱点不明显。

### 2.2 缩略图测试

所有战斗精灵要做三档缩放检查：

| 检查尺寸 | 用途 |
| ---: | --- |
| 100% 原始帧 | 看笔触和边缘 |
| 50% | 模拟游戏内缩放 |
| 25% | 检查剪影是否仍可读 |

25% 时必须还能分辨：

- 玩家头肩。
- 玩家胸口胃纹大色块。
- 武器大方向。
- 敌人类型。
- Boss 阶段。

## 3. 统一视角和体积

### 3.1 角色体积比例

角色不要画成平贴纸片。每个战斗单位都要有：

- 头部顶面。
- 肩膀前后关系。
- 胸腔或躯干厚度。
- 前脚 / 后脚层次。
- 脚底接触点。

### 3.2 朝向定义

四方向命名统一：

| 方向 | 含义 | 画面表现 |
| --- | --- | --- |
| `down` | 面向屏幕下方 | 能看到脸 / 胸口 / 胃纹最多 |
| `up` | 面向屏幕上方 | 能看到背部、后肩、头顶 |
| `left` | 面向屏幕左侧 | 左手、武器、侧胸清楚 |
| `right` | 面向屏幕右侧 | 右手、武器、侧胸清楚 |

不要用一张 `right` 镜像成 `left`，除非这个角色完全左右对称。玩家和持工具敌人不允许纯镜像交付。

### 3.3 影子

每个角色可交付单独影子层，也可以让程序统一画影子。

如果画师交影子：

- 影子透明 PNG。
- 不要烘死在角色帧里。
- 影子椭圆应压扁，偏暖灰。
- 影子中心对齐脚底接触点。

推荐文件名：

```text
shadow_player_ellipse.png
shadow_enemy_small.png
shadow_boss_barn_king.png
```

## 4. 玩家细节需求

### 4.1 无韵回响视觉关键词

玩家必须是战斗画面中最干净、最容易追踪的单位。

关键词：

- 脏银白。
- 暗红胃纹。
- 短刃 / 残片刃。
- 轻盔或骨质外壳。
- 不完整的神性。
- 小体型但姿态稳定。

不要：

- 长白棍。
- 大披风盖住脚。
- 全身碎纹理。
- 和空腹者一样佝偻。
- 胸口胃纹太小。

### 4.2 玩家比例

建议比例：

```text
头部：约 22% 身高
躯干：约 36% 身高
腿部：约 30% 身高
武器：约 35-45px 游戏显示长度
胸口胃纹：游戏内至少 8x8px 可见
```

玩家要比普通敌人更直立、更清楚，但不能像高大骑士。

### 4.3 玩家颜色

建议色组：

| 部位 | 色相 | 明度要求 |
| --- | --- | --- |
| 主体 | 脏银、冷灰白 | 比背景亮 |
| 暗部 | 灰绿、灰蓝、暗褐 | 不要纯黑 |
| 胃纹 | 暗红、酒红 | 必须可读 |
| 武器 | 银白、骨白 | 攻击帧中清楚 |
| 受击 | 暖白 / 淡红闪 | 不要满屏红 |

### 4.4 玩家动画细节

#### idle

4 帧循环。

必须体现：

- 轻微呼吸。
- 胸口胃纹轻微收缩。
- 武器和手不乱晃。
- 脚底不滑。

不要：

- 大幅摇头。
- 身体左右漂移。
- 呼吸导致脚底上下跳。

#### walk

6 帧循环。

要求：

- 脚步交替清楚。
- 身体上下起伏不超过 3px。
- 武器随身体摆动，但不能像漂浮物。
- `left/right` 方向要有前后脚层次。
- `up/down` 方向要能看出迈步，不要只上下抖。

#### attack

首版至少一套通用攻击，推荐三段：

| 段数 | 动作 | 视觉节奏 |
| --- | --- | --- |
| attack_1 | 快斩 | 起手短、出刀快 |
| attack_2 | 横斩 | 刀光更宽 |
| attack_3 | 重斩 | 前倾明显、收招慢 |

每段结构：

1. 起手：身体压低，武器收在一侧。
2. 挥出：刀光最大，武器贴手。
3. 命中：身体前倾，胸口胃纹亮一下。
4. 收招：回到站姿。

武器必须画在手上。刀光可以单独交 `vfx_slash`，但角色帧里仍要能看见手部和武器动作。

#### hit

3 帧。

要求：

- 胸口胃纹暗下去或闭合。
- 身体向受击反方向缩。
- 不要夸张飞出画布。

#### death / collapse

6 帧。

表现：

- 身体失去支撑。
- 胃纹熄灭。
- 少量银灰碎屑。
- 不做血腥尸块。

## 5. 武器和攻击特效

### 5.1 武器设计

武器是“神烬残片”，不是普通剑。

建议造型：

- 短刃。
- 断裂圣器。
- 骨银碎片。
- 仪式刀。

禁止：

- 长白棍。
- 激光剑。
- 现代刀。
- 过长武器遮挡角色。

### 5.2 刀光 VFX

刀光单独交透明 PNG 序列。

| 类型 | 尺寸 | 帧数 | 说明 |
| --- | ---: | ---: | --- |
| `vfx_slash_side` | `192x192` | 6 | 左右攻击 |
| `vfx_slash_down` | `192x192` | 6 | 向下攻击 |
| `vfx_slash_up` | `192x192` | 6 | 向上攻击 |
| `vfx_hit_small` | `128x128` | 4 | 普通命中 |
| `vfx_hit_boss_core` | `192x192` | 6 | Boss 弱点命中 |

刀光风格：

- 银白主色。
- 边缘有暗红胃纹残影。
- 形状短而厚，不要细长激光线。
- 出现快，消失快。

## 6. 敌人细节需求

### 6.1 空腹者

战斗定位：

- 追击玩家。
- 近身咬。
- 死后掉日志碎片。

视觉关键词：

- 干瘪。
- 空碗。
- 喉咙过长。
- 腹部不成比例。
- 破布。

不要：

- 画成普通丧尸。
- 画成骷髅兵。
- 过度血腥。

剪影：

- 上半身前倾。
- 手臂向前够。
- 腿细、步伐急。

攻击前摇：

- 头和肩向后收。
- 嘴 / 碗 / 喉咙区域张开。
- 下一帧前扑。

死亡：

- 不要血爆。
- 身体塌下。
- 从胸口或碗中掉出记忆晶片。

### 6.2 饥民农夫

战斗定位：

- 中距离施法。
- 在玩家路径上播种危险区。

视觉关键词：

- 农夫。
- 种袋。
- 登记牌。
- 旧锄 / 播种器。
- 低头劳动。

不要：

- 像战士挥刀。
- 像法师举法杖。
- 和背景麦草同色。

施法动作：

1. 手伸进种袋。
2. 身体侧转。
3. 撒出种子。
4. 种子落地亮红。

这个动作必须让玩家一眼知道“地上要出危险区”。

### 6.3 饥饿稻草人

战斗定位：

- 精英敌人。
- 释放麦浪 / 地面危险线。

视觉关键词：

- 高瘦。
- 木架。
- 破旧衣服。
- 仪式挂签。
- 枯麦。

不要：

- 太像普通稻草人摆件。
- 太可爱。
- 太细，缩小后看不见。

施法动作：

- 先僵住。
- 手臂或木架打开。
- 背后麦穗形成环。
- 地面出现麦浪预警。

## 7. Boss 谷仓王细节

### 7.1 体型和接地

谷仓王不是普通敌人放大。

必须有：

- 大体积主体。
- 明确底部接地。
- 谷仓门结构。
- 王权符号。
- 胃囊 / 红核弱点。

不要：

- 横版巨大侧脸。
- 只是一栋谷仓贴图。
- 没有脚底或底座。
- 弱点藏在复杂纹理里。

### 7.2 三阶段设计

phase1：

- 仓门紧闭。
- 王冠 / 封条完整。
- 动作慢，压迫感强。

phase2：

- 仓门裂开。
- 内部暗红光露出。
- 木板变形。
- 召唤或吐出危险区时更明显。

phase3：

- 红核暴露。
- 胃囊像不稳定的圣物。
- 弱点亮度高于背景和身体。
- 攻击姿态更前倾。

### 7.3 Boss 攻击图像

| 攻击 | 美术需求 | 玩家应该读到 |
| --- | --- | --- |
| 开仓召唤 | 仓门张开，内部暗红光，碎麦飞出 | 要出小怪 |
| 消化液 | 地面出现暗红圆形预警，随后溅液 | 离开预警圈 |
| 胃室收缩 | Boss 周围四点红圈或裂缝 | 场地收缩 |
| 弱点暴露 | 红核亮，外壳打开 | 现在该打 Boss |

## 8. 房间背景细节

### 8.1 背景分层

每个房间建议交：

```text
bg_xxx_floor.png
bg_xxx_boundary.png
bg_xxx_props_mid.png
bg_xxx_foreground.png
bg_xxx_collision_paintover.png
```

说明：

- `floor`：地面，不透明。
- `boundary`：墙、麦田边界、谷仓边。
- `props_mid`：低矮障碍、农具、木桩。
- `foreground`：前景遮挡，必须透明，不能盖住战斗中心。
- `collision_paintover`：给程序看的碰撞参考，不进游戏。

### 8.2 地面纹理

地面可以有笔触，但不能抢角色。

要求：

- 中心战斗区域低对比。
- 边缘装饰可以更复杂。
- 红色装饰不能太像危险区。
- 黑色裂缝不能太像可走边界。

### 8.3 危险区视觉

危险区必须和背景装饰分开。

危险区用：

- 清晰外圈。
- 半透明填充。
- 0.5 秒以上前摇变化。
- 颜色偏橙红或亮红。

普通装饰红斑用：

- 更暗。
- 无完整圆圈。
- 不发光。
- 不做动画。

### 8.4 房间逐项需求

#### 低语田野入口

用途：

- 第一战斗房。
- 教玩家清怪、采样、掉日志碎片。

画面：

- 开阔路口。
- 枯麦围边。
- 旧木牌。
- 少量谷袋和断农具。

留白：

- 中心至少 `900x420` 清楚可战斗区域。

#### 血肉麦田

用途：

- 引入农夫和危险区。

画面：

- 麦田更密。
- 地面有少量暗红根脉。
- 边缘有旧献祭柱。

注意：

- 不要满屏红。
- 危险区红圈必须比装饰根脉亮。

#### 肠道灌溉渠

用途：

- 当前试玩容易卡 / 误判的房间。

画面：

- 三条渠线要非常清楚。
- 渠线是危险或减速信息，不是背景花纹。
- 玩家路径必须一眼看得出。

建议：

- 渠线边缘画出发亮轮廓。
- 战斗空白区留在渠线之间。
- 不要在中心堆高频纹理。

#### 饥饿谷仓

用途：

- 稻草人精英房。

画面：

- 谷仓门在上侧或远侧。
- 稻草、木梁、挂签。
- 场地中心留给麦浪危险区。

不要：

- 横版立面谷仓。
- 大门遮住战斗中心。

#### 谷仓王胃室

用途：

- Boss 战。

画面：

- 地面环形仪式痕。
- 四周谷仓结构和胃室暗示。
- 中心足够空，Boss 和玩家能绕行。

注意：

- Boss 红核不能和背景红光混在一起。
- 地面仪式线不能像危险区。

## 9. 日志碎片系统美术

日志碎片是核心系统，不是普通掉落。

### 9.1 掉落物

形态：

- 半片脏银晶片。
- 油渍纸页碎片。
- 封蜡痕。
- 微弱蓝灰或银白光。
- 暗红记忆线。

不要：

- 金币。
- 蓝宝石。
- 经验球。
- 普通魔法水晶。

### 9.2 状态

| 状态 | 美术表现 |
| --- | --- |
| 掉落瞬间 | 小碎光从敌人身体散出 |
| 地面等待 | 微微浮动，带小阴影 |
| 可拾取 | 外圈亮一下 |
| 吸收 | 朝玩家胸口或圣匣方向拉成细线 |
| 归档 | 变成纸页上的标签 / 印章 |

### 9.3 UI 关联

战斗掉落物和圣匣 UI 图标必须同源。

也就是说：

- 掉落物是碎片本体。
- UI 图标是碎片被夹进档案后的形态。
- 归档动效要像“样本进入档案”，不是普通拾取。

## 10. 圣匣日志 UI 细节

### 10.1 UI 架构

圣匣日志界面建议分三栏：

1. 左侧：碎片列表。
2. 中间：当前纸页文本。
3. 右侧：样本标签 / 敌人来源 / 胃囊反应。

底部：

- 真相进度条。
- 已归档碎片数量。
- 当前故事拼接状态。

### 10.2 材质

必须出现：

- 油渍纸页。
- 封蜡。
- 金属夹。
- 手写批注。
- 医学档案标签。
- 小型器官样本图标。

不要：

- 赛博蓝面板。
- 透明玻璃 UI。
- 手机 App 卡片。
- 发光霓虹边框。

### 10.3 字体和文字区域

画师不要把正文烘进图里。

需要留给 Godot 文本的区域：

- 纸页正文区。
- 碎片标题区。
- 按钮文字区。
- 进度数字区。

文字底色要保证浅色或暗色文字都能读。

## 11. HUD 细节

### 11.1 生命条

视觉：

- 金属夹 + 暗红液体。
- 不要现代血条。
- 生命减少时可以露出暗纸底。

### 11.2 胃囊四态

| 状态 | 图标需求 |
| --- | --- |
| 开启 | 胃纹张开，淡银红光 |
| 闭合 | 胃纹锁住，颜色暗 |
| 溢血 | 外圈红色过载，边缘喷溅但不血腥 |
| 归档 | 胃纹被封蜡和金属夹固定 |

### 11.3 拾取提示

拾取提示不要像系统弹窗。

建议：

- 小纸条。
- 左侧日志碎片小图标。
- 右侧一行短文字。
- 角落封蜡或夹子。

## 12. 色彩和明度规则

### 12.1 明度层级

建议：

| 层级 | 明度范围 | 说明 |
| --- | --- | --- |
| 玩家主体 | 中高 | 比背景亮 |
| 敌人主体 | 中 | 比背景略亮，但低于玩家 |
| Boss 弱点 | 高 | 战斗焦点 |
| 危险区边缘 | 高 | 必须可读 |
| 背景中心 | 中低 | 不抢角色 |
| 背景边缘 | 可更暗 | 增加氛围 |

### 12.2 色彩禁区

不要让以下颜色互相混淆：

- 玩家胃纹红 vs 危险区红。
- 背景血迹红 vs Boss 弱点红。
- 日志碎片蓝灰 vs 普通 UI 蓝。
- 农夫衣服褐色 vs 地面泥土褐色。

## 13. 动画生产细节

### 13.1 脚底锁定

所有移动动画必须检查脚底：

- 同一方向 walk 循环中，角色不能整体漂移。
- 角色运动由程序移动，不由帧图自己位移。
- 画布中脚底接触点保持稳定。

### 13.2 帧间节奏

不要平均每帧都动一点。动作要有节奏：

- idle：慢。
- walk：均匀。
- attack：起手短、挥出快、收招慢。
- hit：快。
- death：慢一点。

### 13.3 可复用原则

可以复用：

- idle 的部分身体。
- hit 的闪白层。
- death 的碎屑层。
- 左右方向部分笔触结构。

不建议复用：

- 玩家左右攻击整帧镜像。

## 14. 文件检查清单

画师交付前逐项检查：

- PNG 是否透明。
- 画布尺寸是否正确。
- 文件名是否英文小写。
- 帧号是否从 `_0` 开始。
- 同动作同方向帧数是否连续。
- 脚底是否稳定。
- 是否有多余白边。
- 是否有半透明脏边。
- 是否有被裁切的武器或衣角。
- 是否附带源文件。
- 是否附带预览 GIF。

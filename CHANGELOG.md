# 更新日志 / Changelog

这个文件记录项目每次有意义的更新。  
原则：代码可以告诉我们“哪里变了”，但这里要告诉我们“为什么变了”。

## 记录规则

每次提交前，如果改动属于以下任意一种，都要写一条：

- 新功能
- 玩法调整
- 数值调整
- 关卡变化
- 文案更新
- 美术/音效资产接入
- Bug 修复
- 项目结构变化
- 协作文档变化

格式：

```md
## YYYY-MM-DD

### 类型：一句话标题

- 负责人：
- 改动：
- 原因：
- 影响范围：
- 验证：
- 后续：
```

## 2026-06-02

### HUD 素材：改用仓库内原创 SVG 像素占位图

- 负责人：金荣俊 / Codex
- 改动：
  - 将生命、胃囊开启、胃囊闭合、溢血和记忆晶片图标改为项目内原创 SVG 像素占位图。
  - 图标使用直角像素块绘制，可直接作为文本资产进入仓库。
- 原因：
  - 保持远端分支可独立启动，并让占位素材的来源与后续替换路径更清晰。
- 影响范围：
  - `assets/sprites/ui/` 与 `scripts/main.gd`。
- 验证：
  - 已使用 Godot `4.6.3` 后台导入五个 SVG 图标。
  - 已使用 Godot `4.6.3` 无窗口启动检查，脚本与资源加载无错误。
  - 已下载原仓库 `feature/level-hud` 分支快照并独立执行相同检查，远端版本可加载。
- 后续：
  - 正式美术交付后按相同文件语义替换图标。

### 操作手感：补充类摇杆全方向移动

- 负责人：金荣俊 / Codex
- 改动：
  - 保留键盘 `WASD` 与方向键的八方向组合移动。
  - 为移动输入增加手柄左摇杆映射，支持类动作肉鸽的 `360` 度方向控制。
  - 将移动加速与刹停速度提取为独立参数，并保持对角线速度归一化。
- 原因：
  - 试玩反馈认为人物只有四个方向能走，移动显得僵硬；需要更接近类摇杆控制的手感。
- 影响范围：
  - `scripts/main.gd` 中的输入映射与玩家移动响应。
- 验证：
  - 已使用 Godot `4.6.3` 无窗口启动检查，脚本加载无错误。
  - 待键盘和手柄试玩确认。
- 后续：
  - 根据试玩反馈微调加速、刹停和危险区减速比例。

### HUD 表现：基础框体像素化

- 负责人：金荣俊 / Codex
- 改动：
  - 将 HUD 面板从圆角边框改为无抗锯齿直角硬边，并增加块状黑色投影。
  - 为状态面板、目标面板和叙事面板增加暗红或旧银色像素角标。
  - 为生命条和胃囊条增加错位黑色底影，强化层次和像素块感。
- 原因：
  - 首版 HUD 已减少纯文字标记，但基础框体仍偏现代平滑控件，缺少像素风辨识度。
- 影响范围：
  - `scripts/main.gd` 中的 HUD 构建逻辑。
- 验证：
  - 已使用 Godot `4.6.3` 无窗口启动检查，脚本加载无错误。
  - 待本地试玩确认视觉效果。
- 后续：
  - 根据试玩效果决定是否用 `CC0` 像素九宫格贴图替换基础框体。

  - 已使用 Godot `4.6.3` 后台导入首版像素图标。

- 后续：

### Bug 修复：死亡回收后停止战斗实体更新

- 负责人：金荣俊 / Codex
- 改动：
  - 玩家被危险区击杀并回到圣匣后，立即停止当前帧的危险区更新。

- 验证：
  - 已使用 Godot `4.6.3` 无窗口启动检查，脚本加载无错误。
- 后续：
  - 后续实现清房清除危险区时，一并回归测试。
